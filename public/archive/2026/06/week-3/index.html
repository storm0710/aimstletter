arXiv AI에서 확인한 최신 업데이트

논문RAH

RAH는 부모 에이전트가 실행 가능한 스크립트를 생성하여 병렬로 서브 하네스를 스폰하고, 작은 작업에는 구조화된 함수 호출을 활용합니다. 이 접근은 모델 호출 재귀(RLM)와 달리 도구와 실행 환경을 포함한 코드 중심 재귀입니다. 통제된 장문 추론 평가에서 GPT-5 기반으로 Codex 코딩 에이전트 기준을 71.75%에서 81.36%로 개선했고, Claude Sonnet 4.5에서는 89.77%까지 도달했습니다. 향상은 주로 하네스 설계에 기인한다고 보고합니다. 운영적으로는 하위 에이전트의 격리, 파일시스템 접근 제어, 리소스 할당과 모니터링이 중요합니다. 대규모 서브에이전트 생성을 관리하려면 스케줄링·로깅·오류 전파 전략이 필수적입니다.

  • 1. 왜 필요한가요? 장문 추론과 세분화된 병렬 작업에서 모델 호출만으로는 한계가 있어 실행 환경을 포함한 재귀적 구조가 필요합니다.
  • 2. 핵심 구성 요소: 실행 가능한 스크립트 생성기, 서브 하네스 스폰 메커니즘, 파일·코드 실행 도구와 격리 정책입니다.
  • 3. 기존 방식과의 차이점: 단순 모델 재귀와 달리 도구·환경을 포함해 코드 수준에서 재귀를 구현해 성능을 향상시킵니다.
단어 설명
  1. RAH: 실행 도구와 환경을 포함하는 에이전트 하네스를 재귀적으로 생성·운영하는 패턴.
  2. Harness: CI/CD, 배포, 테스트, 관측 등 소프트웨어 전달 과정을 자동화하는 DevOps 플랫폼입니다.
  3. Codex: 코드 작성, 수정, 리뷰 같은 개발 작업을 돕는 OpenAI의 코딩 에이전트 계열 도구입니다.

선별기준 : 해당 주간 수집 데이터에서 날짜, 출처, 업무 적용 가능성을 기준으로 선별된 실제 업데이트입니다.
